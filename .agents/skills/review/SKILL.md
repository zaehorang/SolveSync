# 프로젝트 리뷰

이 skill은 이 프로젝트의 변경사항이 로컬 계약과 규칙을 지키는지 검토할 때 사용합니다.

## 읽어야 할 입력

변경사항을 리뷰하기 전에 다음 문서를 읽습니다.

- `AGENTS.md`
- `docs/ARCHITECTURE.md`
- `docs/ADR.md`

그다음 Git으로 변경된 파일을 확인합니다.

## 체크리스트

다음 항목을 확인합니다.

1. **아키텍처 준수**: 변경된 파일이 `ARCHITECTURE.md`에 정의된 구조를 따르는가?
2. **스택 준수**: 변경사항이 `ADR.md`의 기술 결정 범위 안에 있는가?
3. **테스트**: 새 동작에 대한 테스트가 있는가?
4. **CRITICAL 규칙**: 변경사항이 `AGENTS.md`의 CRITICAL 규칙을 위반하지 않는가?
5. **빌드 가능 여부**: 빌드 명령이 오류 없이 통과하는가?

요청된 리뷰 범위에 맞게 필요한 검증 명령을 실행합니다. 어떤 명령을 실행할 수 없다면 리뷰에 그 사실을 명확히 적습니다.

## 출력

코드 리뷰 요청에서는 구체적인 발견사항을 심각도 순으로 먼저 제시하고 파일/라인 참조를 포함합니다. 발견사항이 없다면 그 사실을 명확히 말합니다.

그다음 아래 체크리스트 표를 포함합니다.

| 항목 | 결과 | 메모 |
| --- | --- | --- |
| 아키텍처 준수 | pass/fail | {detail} |
| 스택 준수 | pass/fail | {detail} |
| 테스트 | pass/fail | {detail} |
| CRITICAL 규칙 | pass/fail | {detail} |
| 빌드 가능 여부 | pass/fail | {detail} |

위반사항이 있으면 구체적인 수정안을 제안합니다.
